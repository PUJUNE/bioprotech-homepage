# Bioprotech Design System

이 디자인 시스템은 **Bioprotech 이사회 발표자료**를 위한 시각 언어와 컴포넌트 가이드를 정의합니다. 기업 신뢰감과 기술적 전문성을 동시에 전달하는 **블루 그라데이션 기반** 디자인이 핵심입니다.

> 출처: 사용자가 제공한 디자인 스펙 문서 (Bioprotech 이사회 발표자료, 2023년 5월). 별도의 코드베이스/Figma는 첨부되지 않았으며, 본 시스템은 PPTX 테마 분석을 정리한 텍스트 사양을 기반으로 재구성되었습니다.

## 1. 브랜드 컨텍스트

**㈜바이오프로테크 (Bio Protech)** — 강원도 원주시 문막읍 동화공단로에 본사를 둔 **의료용 바이오센서 제조 기업**입니다 (설립 2000년 5월 19일, 직원 ~79명). 슬로건: **"Bio sensors & Bio products"**.

**주요 제품군** (실제 IR 자료 기준):
- **ECG Electrode** — 심전도 측정용 1회용 센서 (General / Wetgel / TAB / Neonatal type)
- **Cutaneous Electrode & Unit** — 전기자극기용 패드 및 기기 (Tens)
- **ESU Pencil & Plate** — 전기수술기용 펜슬 및 접지판 (Ergonomic / Standard)
- **Telescopic Smoke Evacuation Pencil** — 연기 흡입 길이 조절식 수술용 펜슬
- **SPO2, Neurology(EEG/EMG)** 등

**조직**: 한국 본사(BPH) + 미국 판매법인(BPU) + 중국 연태(BPC), 광저우(BPG) 생산법인.

**브랜드 톤**: 본 자료는 정기 이사회/IR 발표용 PPT 테마입니다. 화려한 효과보다 **공식적이고 기술적이며 정돈된 기업 보고용** 톤을 유지합니다 — 일관된 L자형 레이아웃, 단일 서체, 두 가지 블루의 그라데이션으로 시각적 정체성을 구축합니다.

## 2. 콘텐츠 파운데이션 (Content Fundamentals)

### 톤 & 보이스
- **공식적, 객관적, 보고체.** 1인칭("저희", "우리")보다는 **명사형 종결**과 **개조식 표현**이 기본입니다.
- "~함", "~임", "~할 예정", "~ 반영 사항" 등 보고서 어조.
- 마케팅적 과장(혁신적인!, 놀라운!) 없음. 사실 기반 기술서술.

### 케이싱 & 표기
- **한·영 혼용**이 자연스럽게 등장합니다 (한국어 본문 + 영문 라벨).
- 영문 회사명/약칭은 **모두 대문자**: `BIO PROTECH`, `BIOPROTECH`, `MEZOO`, `R&D`.
- 사이드바 회사명만 예외적으로 **케이스 보존**: `Bio protech` (브랜드 워드마크처럼 취급).
- 영문 뷰 라벨은 **Title Case**: `Top View`, `Dimension`, `Perspective View`.
- 치수는 영문 + 단위: `Snap distance: 32mm`, `Electrode distance 80mm`.

### 이모지 / 특수문자
- **이모지 사용하지 않음.** 보고용 톤과 어울리지 않습니다.
- 숫자 원형(①②③) 같은 글리프 대신 **그려진 원형 + 숫자 텍스트** 사용.
- 구분선은 ASCII가 아닌 실제 선 도형(`<hr>` 또는 `border`)으로.

### 예시 카피
- 커버 부제: `2023년 5월 정기 이사회`
- 섹션 제목: `Project Overview`, `Design Review`, `Technical Specifications`
- 아젠다 항목 제목: `프로젝트 진행 현황`
- 아젠다 설명: `Q1 마일스톤 달성 및 Q2 추진 계획 보고`
- 콘텐츠 소제목: `Redesign 반영 사항`, `Top View`, `Dimension`

## 3. 비주얼 파운데이션 (Visual Foundations)

### 색상 (Colors)
2색 블루 시스템이 전부입니다. 다른 색은 거의 등장하지 않습니다.

| Token | Hex | 역할 |
|---|---|---|
| `--accent1` | `#5B9BD5` | 미디엄 블루 — 그라데이션 시작점, 인디케이터 |
| `--accent5` | `#4472C4` | 다크 블루 — 그라데이션 종착점, 번호 원형, 섹션 제목 |
| `--dk2` | `#44546A` | 다크 블루-그레이 — 본문 제목, 보조 텍스트 |
| `--bg1` | `#FFFFFF` | 화이트 — 콘텐츠 배경, 사이드바 위 텍스트 |
| `--lt2` | `#E7E6E6` | 라이트 그레이 — 보조 배경 |
| `--lt-blue-gray` | `#D6DCE5` | 도표/차트 보조 배경 |

**그라데이션:** `linear-gradient(180deg, #5B9BD5 0%, #4472C4 65%)`. 방향과 정지점은 절대 변경하지 않습니다.

### 타이포그래피
- **단일 서체**: 맑은 고딕 (Malgun Gothic). 웹 환경에서는 Google Fonts의 **Noto Sans KR**을 대체로 사용합니다 (이 시스템 기본). ⚠️ *원본 폰트 파일이 제공되지 않아 Noto Sans KR로 대체했습니다 — 정확한 매칭이 필요한 경우 Malgun Gothic 라이선스 파일을 추가해 주세요.*
- **두 웨이트만**: Bold (700) + Regular (400). 중간 웨이트(SemiBold, Light) 미사용.
- **위계는 크기와 색상으로**: 같은 Bold라도 `accent5` (블루) vs `dk2` (그레이)로 주/부 구분.

### 배경 (Backgrounds)
- 콘텐츠 슬라이드: **순수 White**.
- 사이드바: **수직 그라데이션** (위쪽이 밝고 아래쪽이 어두운 단방향).
- 커버: **풀블리드 사진 + 80% 알파 그라데이션 오버레이**.
- 패턴/텍스처/일러스트: **사용하지 않음**.

### 깊이 / 그림자
- **drop shadow 일체 미사용.** 깊이감은 색상 대비(그라데이션 vs 화이트)와 반투명 오버레이로만 표현.
- 카드/박스는 `border` 또는 색상 면으로만 구분, `box-shadow` 없음.

### 모서리 (Corner Radii)
- 박스/도형: **0px** (직각). 기업 보고서 톤.
- **유일한 곡선 요소: 아젠다 번호 원형** (`border-radius: 50%`).

### 보더 / 구분선
- 사이드바 내부 장식 수직선: **1-2px White**, 길이 ~280-360pt.
- 커버 회사명-부제 구분선: **2px White**, 길이 ~37pt 짧은 수평선.
- 콘텐츠 영역 내부 분할 보더: 거의 없음. 색면 대비로 분할.

### 애니메이션 / 인터랙션 / 호버
- 정적 발표자료 — **애니메이션 없음**.
- 인터랙티브 프로토타입에서 호버가 필요할 경우: **opacity 0.85** 또는 **`accent5` → 5% 어두운 톤**으로 처리.
- 프레스 상태: **`scale(0.98)`** 정도의 미세한 축소.

### 투명도 / 블러
- 사용 위치 **단 하나**: 커버 슬라이드의 그라데이션 오버레이 (`alpha = 0.80`).
- 블러 효과(`backdrop-filter` 등) **사용하지 않음**.

### 이미지 컬러 무드
- 제품 사진: **중성/쿨 톤**. 흰 배경의 제품 렌더링 또는 의료기기 사진.
- 커버/섹션 디바이더 사진: 그라데이션 오버레이로 인해 **블루 톤으로 통일**됩니다.
- 흑백/그레인 효과 사용 안 함.

### 레이아웃 규칙
- **L자형 레이아웃이 절대 원칙**: 좌측 사이드바(고정 너비) + 우측 콘텐츠 영역.
- 사이드바 너비 (절대 변경 금지):
  - Section Divider = **254pt**
  - Agenda = **198pt**
  - Content = **296pt**
- 콘텐츠 정렬: **좌측 정렬 기본**, 스펙/치수만 **우측 정렬**.
- 슬라이드 사이즈: **960×540pt (16:9)**.

## 4. 아이코노그래피 (Iconography)

이 디자인 시스템은 **아이콘을 거의 사용하지 않습니다**. 정보는 텍스트, 사진, 도형으로만 전달됩니다.

- **아이콘 폰트/SVG 라이브러리: 없음.**
- **이모지: 사용하지 않음.**
- **유니코드 글리프(①②③ 등): 사용하지 않음** — 대신 그려진 원형 + 텍스트 숫자.
- **로고**: 우측 상단 또는 사이드바에 회사 워드마크. (제공된 로고 파일 없음 — `assets/logo-placeholder.svg`로 대체)

### 만약 아이콘이 필요해진다면
공식 가이드는 아니지만, 새로 추가할 경우 다음 원칙을 권장합니다:
- **선 스타일 (line/outline)**, 1.5-2px stroke
- **단색**, `accent5` 또는 `dk2` 컬러
- **24px 그리드** 기준
- CDN: [Lucide Icons](https://lucide.dev/) — 깔끔한 의료/기업 톤과 가장 잘 맞음. (현재 시스템에는 미포함, 사용 시 사용자에게 플래그)

## 5. Do's and Don'ts (요약)

### ✓ Do
- 좌측 그라데이션 사이드바를 모든 콘텐츠 슬라이드에 일관 적용
- `accent1` → `accent5` 그라데이션을 시각적 통일 요소로 사용
- 사이드바에 현재 프로젝트명/세부 주제를 항상 표시 (네비게이션 역할)
- 텍스트 위계는 크기와 Bold/Regular로만 구성
- 영문 회사명/약칭은 대문자 (`BIO PROTECH`, `MEZOO`, `R&D`)
- 기술 스펙/치수는 우측 정렬

### ✗ Don't
- 그라데이션 방향 변경 — `accent1`이 항상 시작점(상단)
- 사이드바 너비를 변경 — Content 296pt, Agenda 198pt, Section 254pt 고정
- `accent1`/`accent5` 외의 테마 색상(오렌지, 그린)을 주요 요소에 사용
- 맑은 고딕(또는 대체 Noto Sans KR) 외 서체 혼용
- drop shadow / 3D 효과 / 둥근 모서리 (아젠다 원형 제외)
- 본문 가운데 정렬 — 좌측 정렬이 기본
- 사이드바 단색 처리 — 그라데이션이 정체성

## 6. 알려진 제한 / 플래그

- ⚠️ **Malgun Gothic 폰트 파일 없음** — Noto Sans KR로 대체했습니다. 픽셀 매칭이 필요한 경우 Malgun Gothic ttf를 `fonts/`에 추가해 주세요.
- ⚠️ **로고 자산 없음** — 텍스트 기반 워드마크 + 플레이스홀더 SVG로 대체했습니다.
- ⚠️ **제품/배경 이미지 없음** — 커버와 섹션 디바이더의 사진은 그라데이션 + 라벨이 있는 플레이스홀더 박스로 대체했습니다.

원본 PPTX 파일이나 이미지 자산을 제공해 주시면 더 정확하게 맞춰드릴 수 있습니다.
